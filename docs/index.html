Опис проєкту

Розроблено Telegram-бота, що допомагає студентам і викладачам швидко знаходити відповіді на найчастіші запитання щодо роботи в Системі дистанційного навчання d-learn.pnu.edu.ua. Бот має зручне меню з категоріями та вміє обробляти довільні текстові запити, повертаючи найрелевантнішу відповідь з бази знань. До багатьох пунктів додаються скріншоти інтерфейсу системи, витягнуті з офіційної методички.

Мета роботи

Спростити пошук інформації про користування Системою дистанційного навчання, зменшити навантаження на Центр дистанційного навчання та надати студентам і викладачам уніфіковане джерело відповідей у звичному середовищі — Telegram.

Можливості бота

📋 Готові категорії

Студентам, викладачам, методична інструкція та корисні посилання — усе у головному меню.

🔍 Пошук вільним текстом

Користувач пише запитання своїми словами — бот шукає найрелевантнішу відповідь.

🖼️ Скріншоти зі сторінок методички

До багатьох тем додаються зображення зі сторінок офіційного PDF-посібника.

🤔 Розумні підказки

Якщо точної відповіді немає, бот пропонує найближчі альтернативи та офіційні джерела.

✏️ Простий формат бази

Уся база знань — один JSON-файл. Додати нове запитання займає декілька секунд.

🛡️ Безпечне зберігання токена

Токен Telegram зберігається у файлі .env, що виключений з репозиторію.

Використані технології

  • Node.js 20+ — середовище виконання
  • grammY — фреймворк для Telegram Bot API
  • Fuse.js — нечіткий пошук по тексту
  • dotenv — конфігурація через .env
  • pdfjs-dist — парсинг PDF-методички
  • @napi-rs/canvas — рендеринг сторінок у PNG
  • JSON — формат бази знань
  • Git / GitHub — контроль версій

Архітектура

Проєкт організований за принципом «модульного бота»:

tg-bot-university/
├── src/
│   ├── bot.js                 точка входу, команди /start, /help, /menu
│   ├── handlers/
│   │   ├── menu.js            побудова інлайн-клавіатур
│   │   ├── faq.js             відправка відповіді з фото
│   │   └── search.js          обробка вільного тексту
│   └── search/index.js        конфігурація Fuse.js та пошук
├── data/faq.json              база знань: 55+ пунктів з 4 категорій
├── assets/images/             89 скріншотів сторінок методички
├── scripts/extract-images.js  одноразовий скрипт витягу зображень з PDF
└── docs/                      сайт-презентація проєкту

Уся база знань зберігається у структурованому файлі data/faq.json: для кожного запитання вказано категорію, заголовок, повний текст відповіді, ключові слова для пошуку та шляхи до прив'язаних зображень.

Як працює пошук вільним текстом

  1. Запит користувача нормалізується (нижній регістр, прибирання лапок).
  2. Розбивається на токени-слова довжиною від 3 символів.
  3. Fuse.js шукає збіги у трьох полях кожного пункту бази:
    • question — заголовок (вага 50%)
    • keywords — ключові слова (вага 35%)
    • answer — повний текст відповіді (вага 15%)
  4. Бали з пошуку по цілій фразі та по окремих токенах сумуються для кожного пункту.
  5. Якщо найкращий бал ≥ 0,55 — відправляється відповідь. Якщо ≥ 0,35 — пропонуються підказки. Інакше — повідомлення про відсутність відповіді з посиланнями на офіційні джерела.

Алгоритм коректно обробляє опечатки, відмінки та перефразування українською мовою, що було перевірено на десятках тестових запитів.

Етапи виконання роботи

  • Аналіз джерел інформації: офіційний FAQ та методичні вказівки (PDF, 89 сторінок).

  • Реалізація головного меню, підменю категорій та обробників натискань.
  • Інтеграція Fuse.js для пошуку по вільному тексту з токенізацією.
  • Реалізація відправки одиничних фото та медіа-груп з підписами.
  • Тестування на реальних запитах (опечатки, парафрази, відмінки).
  • Оформлення документації та сайту-презентації проєкту.
